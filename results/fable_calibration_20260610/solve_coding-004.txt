Implement a Python class `LRUCache` with O(1) time complexity for both `get` and `put` operations. The cache has a fixed capacity set at construction.

```python
class LRUCache:
    def __init__(self, capacity: int): ...
    def get(self, key: int) -> int: ...  # return -1 if not found
    def put(self, key: int, value: int) -> None: ...
```

Constraints and edge cases your implementation must handle correctly:
- `capacity` is always >= 1
- `put` on an existing key updates the value AND marks it as most recently used
- `get` on an existing key marks it as most recently used
- When the cache is at capacity and a new key is inserted, evict the LEAST recently used key
- The following sequence must produce exactly these results:

```python
c = LRUCache(2)
c.put(1, 1)
c.put(2, 2)
c.get(1)       # returns 1 (marks 1 as recently used)
c.put(3, 3)    # evicts key 2 (not key 1, because 1 was just accessed)
c.get(2)       # returns -1 (evicted)
c.put(4, 4)    # evicts key 1 (3 is more recent from the put)
c.get(1)       # returns -1 (evicted)
c.get(3)       # returns 3
c.get(4)       # returns 4
c.put(3, 30)   # updates value of 3 (marks as most recent)
c.put(5, 5)    # evicts key 4 (3 was just touched by put)
c.get(4)       # returns -1 (evicted)
c.get(3)       # returns 30
```

Do NOT use `functools.lru_cache` or `collections.OrderedDict`. Implement the data structure from scratch using a dict and a doubly linked list.
